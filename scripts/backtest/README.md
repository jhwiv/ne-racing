# Backtest Harness

Offline measurement of the Railbird advice engine against historical race
results. Designed to:

1. **Compare engine versions head-to-head.** v1 (heuristic, currently in
   production via `index.html`) and v2 (methodology-fixed, in
   `scripts/lib/scoring.js`) run against the same races and produce side-by-side
   metrics.
2. **Compare both against a baseline.** The morning-line implied distribution
   is included as `baseline_ml`.
3. **Degrade gracefully** when no result data is available. The harness will
   still score every race and print a clear "outcome metrics unavailable" notice.

## Quick start

```bash
# Run against everything on disk
node scripts/backtest/run.js

# Include placeholder fixtures (useful for end-to-end demo / development)
node scripts/backtest/run.js --include-fixtures

# Only races that have results
node scripts/backtest/run.js --require-results

# Filter by track or date range
node scripts/backtest/run.js --track SAR --from 2025-07-10 --to 2025-09-01

# Persist the full JSON report
node scripts/backtest/run.js --out /tmp/backtest.json

# Compare only specific engine versions
node scripts/backtest/run.js --versions v2
```

## Data sources

`load_corpus.js` reads, in priority order:

| Priority | Path                                | Source                                  |
| -------- | ----------------------------------- | --------------------------------------- |
| 1        | `data/normalized/{year}/{track}/`   | Output of `scripts/ingest/run.js`       |
| 2        | `data/entries-{TRACK}-{DATE}.json`  | Hand-curated NYRA cards                 |
| 3        | `data/fixtures/`                    | Placeholder fixtures (opt-in)            |

When the same race id appears in multiple sources, **the copy with results
wins** regardless of source. Otherwise normalized > entries > fixtures.

A race "has results" when its `results.finish_positions[]` field is non-empty
and at least one row has `position === 1` (the winner) with a `pp`. The
optional `win_payout` enables flat-bet ROI.

## Metrics

| Metric                  | Lower/Higher better | Notes |
| ----------------------- | ------------------- | ----- |
| `log_loss_mean`         | Lower               | Per race, on the winner. ML baseline for thoroughbreds is typically 1.6–2.2. |
| `brier_mean`            | Lower               | Multi-class Brier across all horses; bounded above by ~2. |
| `top1_rate`             | Higher              | Fraction of races where model's #1 pick won. |
| `top3_rate`             | Higher              | Fraction where the winner was in model's top 3. |
| `flat_top_pick_roi_pct` | Higher              | $2 flat win on every #1 pick. Requires `win_payout`. |
| `overlay_bets_*`        | Higher              | $2 flat win on every horse with overlay > 8% and score ≥ 55. |
| Calibration buckets     | `|Δ|` lower         | Predicted prob vs realized hit rate, deciled. |

## Known limitations

- **No real results are committed to this repo.** The `data/fixtures/with_results/`
  file generated by `_generate_synthetic_results.js` draws winners from the
  ML implied distribution; it is for pipeline-validation only and is clearly
  marked `synthetic_results: true` in its meta block. Real Saratoga/Aqueduct
  results require connecting `scripts/ingest/theracingapi_adapter.js` with a
  subscription, or hand-curating from Equibase charts.
- **Single-card-level metrics.** No multi-race exotics yet (Pick 3/4/5/6).
- **No walk-forward refit.** v2 weights are still hand-tuned; the conditional-
  logit refit will land in PR #2 alongside a true train/validation split.
- **No bankroll dynamics.** ROI is flat-stake; Kelly + fractional-Kelly
  simulation comes after PR #2.

## Adding real results

Drop a JSON file at `data/normalized/{year}/{track}/{date}.json` shaped like:

```json
{
  "races": [
    {
      "id": "SAR-20250710-R1",
      "track": "SAR",
      "date": "2025-07-10",
      "num": 1,
      "type": "AOC",
      "horses": [ /* … */ ],
      "results": {
        "finish_positions": [
          { "pp": 4, "horseName": "Caffery Park", "position": 1, "win_payout": 6.40 }
        ]
      }
    }
  ]
}
```

The backtest will pick it up automatically on the next run.
